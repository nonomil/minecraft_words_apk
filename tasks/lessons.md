# 历史教训 (Lessons Learned)

```
- [日期] [场景] → 错误根因 → 下次防范方式
```

## 教训记录

- [2026-03-04] 大型代码库扫描 → 本地 extract 零成本提取代码结构效率高 → 优先使用 extract 而非直接 Codex 扫描
- [2026-03-04] 大型代码库扫描 → 单模块统计（extract）与逻辑划分（21个模块）存在差异 → 需人工判断实际模块数，不能只看物理文件统计
- [2026-03-04] 大型代码库扫描 → 纯 JavaScript 项目缺少类型信息，函数签名需推测 → TypeScript 项目扫描质量更高
- [2026-03-04] 大型代码库扫描 → 策略 B（CC 直接执行）可作为 Codex 不可用时的兜底方案 → 基于 extract-data.json 生成结构化文档
