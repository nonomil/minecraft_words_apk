# vA 逐小时执行清单（4-6小时）

> 用法：从上到下执行。每个时间块只做该块内任务，完成后打勾。

## 0. 开始前（10分钟）

- [ ] 进入目录：`mario-minecraft-game_V1/apk`
- [ ] 检查环境：`node -v && npm -v`
- [ ] 检查工作区：`git status`
- [ ] 建分支：`git checkout -b feature/vA-tts-save-stats`

通过标准：
1. 分支切换成功。
2. 环境命令可执行。

## 1. 第1小时：设置页“测试发音”按钮（UI + 绑定）

### 1.1 UI（20分钟）

文件：`Game.html`

- [ ] 在设置区增加按钮：`id="btn-speech-test"`
- [ ] 文案：`测试发音`
- [ ] 位置：朗读相关设置附近

自检：
- [ ] 页面可见该按钮
- [ ] 移动端布局未明显错位

### 1.2 事件绑定（30分钟）

文件：`src/modules/16-events.js`

- [ ] 在 `wireSettingsModal()` 读取 `btn-speech-test`
- [ ] 点击逻辑：
  - [ ] 若 `settings.speechEnabled` 关闭，toast 提示
  - [ ] 否则调用 `speakWord({ en: "hello", zh: "你好" })`
  - [ ] 异常捕获并提示失败原因

### 1.3 快速验证（10分钟）

- [ ] 运行项目：`npm run dev`
- [ ] 手动点击按钮，确认“有发音或有错误提示”

通过标准：
1. 按钮存在。
2. 点击有明确反馈。

## 2. 第2小时：档案页“备份/恢复存档”快捷入口

### 2.1 档案页加按钮（25分钟）

文件：`Game.html`

- [ ] 在 `#profile-modal .profile-actions` 中新增：
  - [ ] `#btn-profile-export-save`
  - [ ] `#btn-profile-import-save`

### 2.2 绑定逻辑（25分钟）

文件：`src/modules/08-account.js`

- [ ] 在 `wireProfileModal()` 中获取新按钮
- [ ] 导出按钮绑定 `handleExportSave`
- [ ] 导入按钮绑定 `handleImportSave`
- [ ] 保留原保存弹窗中的旧入口（不删除）

### 2.3 快速验证（10分钟）

- [ ] 打开档案页，确认两个按钮显示
- [ ] 点击可触发导出/导入流程

通过标准：
1. 档案页可直接备份/恢复。
2. 旧入口仍可用。

## 3. 第3小时：学习统计“正确率”

### 3.1 统计逻辑（35分钟）

文件：`src/modules/08-account.js`

- [ ] 在 `renderLearningStats(account)` 中聚合 `wordStats`：
  - [ ] `totalCorrect`
  - [ ] `totalWrong`
  - [ ] `accuracyPercent`
- [ ] 新增统计卡：`正确率 xx%`
- [ ] 无答题记录时显示 `0%`

### 3.2 快速验证（25分钟）

- [ ] 打开档案页学习统计
- [ ] 核对正确率显示
- [ ] 验证无记录时不报错

通过标准：
1. 正确率稳定显示。
2. 没有 NaN/undefined。

## 4. 第4小时：测试与回归（核心）

### 4.1 自动化（30分钟）

- [ ] 执行：
```bash
npx playwright test -c tests/e2e/playwright.config.mjs tests/e2e/specs/opt-0226-task1-save-transfer.spec.mjs tests/e2e/specs/opt-0226-task5-learning-stats.spec.mjs
```

### 4.2 手工回归（20分钟）

- [ ] 设置页测试发音
- [ ] 档案页备份/恢复
- [ ] 正确率显示
- [ ] 主流程：进入游戏 -> 暂停 -> 继续 -> 存档

### 4.3 问题收敛（10分钟）

- [ ] 记录失败点
- [ ] 修复后重跑最小回归

通过标准：
1. 自动化通过或有明确失败原因记录。
2. 手工4项通过。
