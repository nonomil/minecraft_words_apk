# 统一测试回归与发布闸门

## 一、回归分层

1. 冒烟：启动、进入游戏、暂停/继续、存档
2. 功能：本版本新增功能逐项验证
3. 回归：历史核心功能不退化

## 二、建议测试矩阵

### vA 完成后

1. 设置页测试发音
2. 档案页备份/恢复
3. 学习统计正确率

### vB 完成后

1. 趋势/周对比渲染
2. 成就进度变化
3. 音效开关有效

### vC 完成后

1. TTS 诊断可用
2. 自检入口可用
3. SW 注册 + 版本缓存可用

### vD 完成后

1. Provider 合同一致
2. target 切换有效
3. Web/APK 不回归

## 三、发布闸门（全部满足）

1. 本版本功能验收全部通过
2. 关键测试全部通过（自动化 + 手工）
3. 回滚方案已准备
4. 文档已更新（计划、变更、已知问题）

## 四、发布后48小时观察

1. 记录发音失败反馈数
2. 记录存档相关反馈数
3. 记录退款与售后耗时
4. 若异常上升，按版本回滚点处理
