# 词库治理执行计划（执行版）

日期：2026-02-18  
范围：`js/vocabularies/**` 与 `android-app/web/js/vocabularies/**` 的上游源词库治理

## 1. 目标

1. 建立统一词库质量门禁（可重复执行）。
2. 输出可落地的补词/删词候选清单。
3. 将“词条质量 + 图片质量 + 覆盖率”纳入持续流程。

## 2. 执行阶段

### 阶段 A：基线抽取
- 运行 `npm run vocab:extract`
- 产物：词条总量、唯一词数量、重复词清单、文件级统计

### 阶段 B：质量校验
- 运行 `npm run vocab:validate`
- 产物：空中文、空图片、坏 URL 模式等问题清单
- 输出按 P0/P1 分级

### 阶段 C：外部覆盖率比对
- 运行 `npm run vocab:bench`
- 对比通用词频与 Minecraft 术语来源
- 输出覆盖率与缺失词样本

### 阶段 D：治理动作
- P0：先修复空中文、冲突重复、关键坏链接
- P1：补图、URL 修复、分层调整（基础/进阶）
- P2：基于覆盖率做增删优化

## 3. 通过标准

1. P0 问题清零。
2. 图片有效性问题持续下降（每轮可量化）。
3. 核心词表覆盖率持续提升，并有变更记录。

## 4. 周期

- 每次词库改动后执行 `npm run vocab:audit`
- 每周至少一次完整审计并更新报告
