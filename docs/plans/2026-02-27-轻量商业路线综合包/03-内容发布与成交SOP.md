# 内容发布与成交SOP（轻量版）

## 1. 内容发布SOP（小红书）

### 1.1 单条内容标准结构

1. 前3秒：冲突点
- 例：`“孩子爱打游戏，不爱背单词？”`

2. 中段：机制展示
- 展示“必须答题才能过关”“答错复盘”等画面

3. 结尾：结果与行动
- 展示学习统计或复盘页
- 引导：`“想看试玩，私信我关键词：单词冒险”`

### 1.2 内容类型配比（每周3条）

1. 机制型 1 条（展示玩法）
2. 反馈型 1 条（展示用户评价/学习结果）
3. FAQ 型 1 条（答疑：安装、安全、适龄）

## 3. 咸鱼商品SOP

### 3.1 商品标题

- 关键词：`英语学习`、`单词`、`闯关`、`无广告`
- 示例：`英语单词闯关学习游戏（安卓）无广告，边玩边记词`

### 3.2 商品详情必须包含

1. 适用人群
2. 核心机制（强制答题）
3. 交付内容（下载+安装教程+售后）
4. 免责说明（设备兼容差异）

## 4. 售后SOP（减少退款）

### 4.1 三步排障顺序

1. 安装权限检查
2. 网络与存储空间检查
3. 版本与机型兼容说明

### 4.2 高概率问题预设话术

1. 安装失败：引导未知来源权限、空间清理
2. 发音异常：引导语音权限、首次点击解锁音频
3. 进度迁移：指导使用存档码导入导出

## 5. 每周优化动作

1. 汇总”咨询最多的3个问题”，下周内容直接回答。
2. 替换点击率最低的封面模板。
3. 更新商品详情中的案例图，避免素材老化。

## 审查意见（2026-02-27）

### 售后 SOP 与代码现状的核对

**4.2 发音异常处理：说法准确，但需补充**

- 代码中 `03-audio.js` 的 `wireAudioUnlock()` 确实要求用户首次交互（点击/触摸/按键）才能解锁音频，这是浏览器/WebView 的强制限制。”首次点击解锁音频”这个说法准确。
- 补充：TTS 有三层 fallback（Web Speech API → Capacitor 原生 TTS → Google 在线 TTS）。如果用户反馈”完全没有声音”，排查顺序应为：① 系统音量 → ② 语音权限 → ③ 网络（在线 TTS fallback 需要网络）→ ④ 机型兼容性（Android 7 以下 Web Speech API 可能缺失）。建议将此排查顺序加入售后话术。

**4.2 进度迁移：功能已实现，但 UX 是瓶颈**

- 存档码导出/导入已在 `08-account.js` 中实现（`handleExportSave` / `handleImportSave`）。
- 当前 UX 问题：导入使用原生浏览器 `prompt()` 对话框，在部分 Android WebView 中可能被拦截或显示异常。建议在售后话术中说明”如果弹窗没有出现，请尝试重启游戏后再操作”。
- 存档码入口在设置页内，用户不容易发现。售后时需主动告知入口位置。

### 私信 SOP 补充建议

- 首次回复模板中建议加一句：”**安卓手机可用，iOS 暂不支持**”，避免 iPhone 用户下单后退款。
- 成交后交付模板中建议明确 APK 版本号，方便后续用户反馈时定位问题。

### 咸鱼商品标题关键词建议

- 当前示例标题含”英语学习”、”单词”、”闯关”、”无广告”，方向正确。
- 补充可测试的关键词：`小学英语`、`人教版单词`、`游戏学英语`、`不背单词过不了关`。
- 避免使用：`Minecraft`、`马里奥`、`我的世界`（平台版权审核风险）。
