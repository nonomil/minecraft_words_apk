# vA 运营前修正与最小开发计划（P0，落地版）

## 目标

在开始持续引流前，完成“文案风险修正 + 三项最小功能”，把售后高频问题提前处理掉。

## 周期

1. Day 1：文案修正 + 素材统一。
2. Day 2：三项最小开发 + 回归。

## Day 1（非代码，2-3小时）

### Task A1：截图口径统一为6张

1. 输入：现有 `01`、`02` 文档。
2. 操作：把“8张截图”全部改为“6张核心截图”。
3. 输出：统一截图清单（固定6张）。
4. 验收：全文不再出现“8张”。

建议命令：
```bash
rg -n "8张|8 张" docs/plans/2026-02-27-轻量商业路线综合包
```

### Task A2：咸鱼标题去IP词

1. 输入：SOP 文档。
2. 操作：明确禁用词和替代词模板。
3. 输出：可直接复制的标题模板（不含 IP 词）。
4. 验收：模板区无 `Minecraft/马里奥/我的世界`。

### Task A3：兼容与平台预筛选文案

1. 固定写入：`建议 Android 8.0 及以上`。
2. 固定写入：`安卓手机可用，iOS 暂不支持`。
3. 验收：商品详情模板、首条私信模板都出现上述文案。

## Day 2（代码，3-5小时）

### Task B1：设置页“测试发音”按钮

文件：
1. `Game.html`
2. `src/modules/16-events.js`
3. `src/modules/12-challenges.js`

执行步骤：
1. 在设置面板新增 `#btn-speech-test`。
2. 绑定点击事件，触发 `speakWord({ en: "hello", zh: "你好" })`。
3. 失败时 toast 提示（如“请先开启朗读”）。

验收：
1. 点击按钮有实际发音或明确失败提示。

### Task B2：档案页“备份/恢复存档”快捷入口

文件：
1. `Game.html`
2. `src/modules/08-account.js`

执行步骤：
1. 在档案弹窗增加两个按钮：备份存档码、恢复存档码。
2. 绑定到 `handleExportSave` 和 `handleImportSave`。
3. 保留原入口，避免功能回退。

验收：
1. 档案页可直接导出/导入，不必绕到保存弹窗。

### Task B3：学习统计增加“正确率”

文件：
1. `src/modules/08-account.js`

执行步骤：
1. 汇总 `wordStats` 的 `correct` 与 `wrong`。
2. 新增统计卡：`正确率 xx%`。
3. 无记录时显示 `0%`。

验收：
1. 档案页学习统计显示正确率。
