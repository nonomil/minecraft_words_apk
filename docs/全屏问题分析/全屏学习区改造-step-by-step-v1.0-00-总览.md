# 全屏学习区改造 Step-by-Step 多版本总览

日期：2026-02-18
关联基线：`全屏学习区改造-最终实施计划.md`

## 1. 版本说明

1. `v1.0-稳健推进版`（推荐）：复用现有移动链路，分阶段收敛风险。
2. `v1.1-快速交付版`：优先交付可用全屏学习体验，压缩里程碑。
3. `v2.0-强化治理版`：在稳健版基础上增加清理与治理动作，适合中长期维护。

## 2. 选择建议

1. 时间紧（1-2天出可测包）：选 `v1.1`。
2. 追求稳定上线（3-4天）：选 `v1.0`。
3. 计划长期迭代（4-6天）：选 `v2.0`。

## 3. 文件索引

1. `全屏学习区改造-step-by-step-v1.0-稳健推进版.md`
2. `全屏学习区改造-step-by-step-v1.1-快速交付版.md`
3. `全屏学习区改造-step-by-step-v2.0-强化治理版.md`

## 4. 统一实施注意（非里程碑硬约束）

1. `mobile-app.js` 的 `resize` auto-switch 逻辑使用 `window.innerWidth > 768` 可能在横屏手机误判为桌面。
2. 实施时建议在原生环境下跳过该 auto-switch，或改为更稳健的判定策略（例如短边阈值）。
3. 该项归入代码实施注意，不单列为计划验收项。
