### Step 3：实现 `pushPause` / `popPause`（`13-game-loop.js` 顶部）

在全局变量区新增：

```js
let pauseStack = 0;

function pushPause() {
    pauseStack++;
    paused = true;
}

function popPause() {
    pauseStack = Math.max(0, pauseStack - 1);
    if (pauseStack === 0) paused = false;
}
```

同时保留 `paused` 变量本身不变（其他模块仍可读取 `paused` 判断状态）。

### Step 4：替换背包弹窗（`13-game-loop.js:909/922`）

```js
// 改前
function showInventoryModal() {
    pausedByModal = !paused;
    paused = true;
}
function hideInventoryModal() {
    if (pausedByModal) paused = false;
    pausedByModal = false;
}

// 改后
function showInventoryModal() {
    pushPause();
}
function hideInventoryModal() {
    popPause();
}
```

### Step 7：修复挑战失败重试的 setTimeout 竞态（`12-challenges.js:627`）

当前问题：答错后 2.5s `setTimeout` 到期时强制恢复 `paused`，若此时背包已打开则产生冲突。

```js
// 改前
setTimeout(() => {
    paused = resumePausedState;  // 强制覆盖
    startLearningChallenge(...);
}, 2500);

// 改后
setTimeout(() => {
    popPause();  // 只弹出挑战自己压入的那一层
    startLearningChallenge(...);
}, 2500);
```

同时在 `startLearningChallenge` 开头改为 `pushPause()`，在 `completeLearningChallenge` 正常结束路径改为 `popPause()`。

**多弹窗叠加场景（最关键）：**

- [ ] 词库提示弹窗正常关闭后游戏恢复

**边界场景：**
- [ ] 快速连续打开关闭背包 10 次，`pauseStack` 不累积（不出现永久暂停）
