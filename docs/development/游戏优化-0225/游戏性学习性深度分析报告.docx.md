| 🔴 立刻修  ·  G-1  卡住检测 Bug：\_lastStuckX 每帧更新，45 帧计数器永远无法触发 |
| :---- |
| *📁 13-game-loop.js · update() 卡住检测段* |
| if (hasInput && Math.abs(player.x \- player.\_lastStuckX) \< 0.5 && player.grounded) {   player.\_stuckFrames++;   if (player.\_stuckFrames \> 45\) { /\* 强制解卡 \*/ } } else {   player.\_stuckFrames \= 0; } player.\_lastStuckX \= player.x;  // ← 每帧都更新！ |
| \_lastStuckX 在 if/else 判断完成之后立即更新。 这意味着下一帧的比较是「当帧位移」而不是「累积静止时间」。 只要有输入但确实没动，每帧 \_stuckFrames++ 会正常累积…… 但实际上 player.x 可能每帧有极小的浮动（0.3-0.4px）， 一旦某帧 |Δx| ≥ 0.5（物理计算误差），\_stuckFrames 立刻归零， 导致实际上 45 帧的条件极难持续触发。孩子卡墙时解卡失效。 |
| **✅ 建议修复** 把 \_lastStuckX 更新移到「确认在移动」的分支里： if (hasInput && player.grounded) {   if (Math.abs(player.x \- player.\_lastStuckX) \< 0.5) {     player.\_stuckFrames++;     // ← 不更新 \_lastStuckX，保持记录卡住起始位置   } else {     player.\_stuckFrames \= 0;     player.\_lastStuckX \= player.x;  // 正常移动时才更新   } } else {   player.\_stuckFrames \= 0;   player.\_lastStuckX \= player.x; } |
