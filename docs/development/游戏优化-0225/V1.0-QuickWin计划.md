### Step 6：G-1 卡住检测 Bug（`13-game-loop.js`）

定位卡住检测段（约 `:334`），重写为：

```js
if (hasInput && player.grounded) {
    if (Math.abs(player.x - player._lastStuckX) < 0.5) {
        player._stuckFrames++;
        // 不更新 _lastStuckX，保持记录卡住起始位置
    } else {
        player._stuckFrames = 0;
        player._lastStuckX = player.x;  // 正常移动时才更新
    }
    if (player._stuckFrames > 45) {
        player.y -= blockSize * 0.8;
        player.velY = -2;
        player._stuckFrames = 0;
        player._lastStuckX = player.x;
    }
} else {
    player._stuckFrames = 0;
    player._lastStuckX = player.x;
}
```

### Step 8：护甲减伤重新设计（`13-game-loop.js`）✅ 已完成

重新设计各护甲减伤比例，参考 Minecraft 原版护甲值：

| 护甲 | 旧减伤 | 新减伤 | 说明 |
|------|-------|-------|------|
| leather | 0.20 | 0.25 | 皮革，入门 |
| chainmail | 0.35 | 0.40 | 锁链，中等 |
| iron | 0.55 | 0.55 | 铁甲，不变 |
| gold | 0.65 | 0.35 | 黄金防御弱于铁（原版设定） |
| diamond | 1.00 | 0.75 | 强但不无敌 |
| netherite | 1.00 | 0.82 | 最强，仍会掉血 |

`minDamage` 从 `hasArmor ? 0 : 1` 改为固定 `1`，任何护甲下至少受 1 点伤害。
