# 小程序拓展分析与架构决策（Web / PWA / APK / 小程序）

- 日期：2026-02-27
- 目标：回答“APK 已可 TTS 发音，是否还要打通网页版本，以及如何为小程序迁移做准备”

## 1. 结论（先看这个）

建议继续打通网页版本 TTS，原因不是“多做一个端”，而是建立跨端统一语音能力基线，为小程序迁移降风险。

推荐路线：

1. `v1.22.0`：先完成 Web 端 TTS 稳定化 + PWA 可用性基线（权限/缓存/兜底）
2. `v1.23.0`：抽象统一 TTS Provider，接入小程序 Provider（微信/uniapp 对应 API）
3. `v1.24.0`：做班级场景增强（词库分发、账号同步、课堂弱网容错）

## 2. 现状判断

### 2.1 当前能力

- APK：TTS 已可用（Capacitor 原生插件路径可行）
- Web：已有 `speechSynthesis` + 在线兜底代码路径，但线上存在“有时不发音”风险
- PWA：有 `manifest` 与 `service-worker.js` 文件，但稳定性与缓存更新策略需要加强
- 小程序：不能直接复用浏览器 `speechSynthesis`，必须接小程序原生语音接口

### 2.2 核心差异

- APK 是“原生容器 + JS 逻辑”，权限和 TTS 插件可控
- Web/PWA 受浏览器策略影响（手势触发、自动播放限制、后台行为）
- 小程序运行时与浏览器不同，TTS 调用模型不同

## 3. 为什么 Web 版要调通

1. 小程序迁移前置验证：先把“纯 JS 业务层 + 语音抽象层”分离，减少后续重构。
2. 降低回归成本：APK、Web 都可跑时，问题能快速定位到“业务逻辑 vs 端能力”。
3. PWA 场景价值：课堂内无需安装 APK，可直接扫码打开，尤其适合轻部署。

## 4. PWA 是什么（结合本项目）

PWA（Progressive Web App）是“可安装网页应用”：

- 用户可“添加到桌面”
- 通过 Service Worker 做静态资源缓存，支持一定离线能力
- 仍然是 Web，不等于原生 App，语音/后台等能力受浏览器限制

对本项目的现实意义：

1. 快速分发（课堂/家长端打开成本低）
2. 更新灵活（无需应用商店）
3. 但不能把它当作 APK 完全替代

## 5. 方案对比

### 方案 A：各端各写（不抽象）

- 优点：短期改动快
- 缺点：后续维护成本高，功能漂移快，回归复杂
- 结论：不推荐

### 方案 B：统一业务层 + TTS Provider 抽象（推荐）

- 优点：业务逻辑不关心平台，端差异收敛在 Provider
- 缺点：初期需要一次架构整理
- 结论：推荐，最平衡

### 方案 C：全部改为云端 TTS（服务化）

- 优点：声音一致、可运营统计
- 缺点：网络依赖、成本、延迟、合规
- 结论：可作为 B 的增强，不建议第一阶段就做

## 6. 推荐架构（目标形态）

统一接口：

```text
speak(text, lang, options) -> Promise<Result>
stop() -> Promise<void>
isAvailable() -> boolean
diagnose() -> { provider, reason, detail }
```

Provider 分层：

1. `apkNativeProvider`（Capacitor TextToSpeech）
2. `webSpeechProvider`（speechSynthesis）
3. `miniProgramProvider`（小程序语音 API）
4. `cloudAudioProvider`（可选，在线音频兜底）
5. `nullProvider`（不可用时，返回可诊断错误）

## 7. 成功标准（验收口径）

### 功能标准

1. 同一个词条在 APK / Web / 小程序三端都可触发发音
2. 失败时能给出明确诊断（权限、网络、策略、Provider 不可用）
3. 发音设置（开关、语速、语言）在三端行为一致

### 稳定性标准

1. 首次进入后 3 次内触发发音成功率 ≥ 99%（可观测）
2. 缓存更新后不会出现“旧脚本导致发音失效”
3. 弱网下能自动降级，不阻塞主流程

## 8. 风险与应对

1. 浏览器策略变更导致 Web 发音异常  
应对：保留多 Provider 兜底 + 诊断面板

2. 小程序端 API 差异（微信/uni-app/字节）  
应对：Provider 层抽象 + 平台实现隔离

3. 资源缓存导致版本错乱  
应对：版本号驱动缓存键 + 升级提示
