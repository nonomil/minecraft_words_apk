## 一、根本原因总结

优化提交（8952e75）引入了 **3 类问题**，导致 APK 无法正常进入游戏和弹窗异常：

| 优先级 | 问题 | 影响 |
|--------|------|------|
| P0 | `09-vocab.js` / `08-account.js` 含 UTF-8 BOM | APK WebView 可能解析失败，导致关键函数未定义 |
| P0 | `Game.html` 编码损坏（已修复） | 所有中文 UI 乱码，弹窗文字不可读 |
| P1 | `btn-export-save` / `learning-stats-panel` 缺失 | 存档导出按钮无效，学习统计面板空白 |
| P1 | `normalizeWordEntry` 跨文件依赖 | 若 `09-vocab.js` 加载失败，挑战完成时 ReferenceError |
| P2 | 挑战答错无限重试 + 输入锁 | 答错后 input 锁定 2.5s 再重试，体验差但不崩溃 |

## 二、详细问题分析

### 问题 1（P0）：`09-vocab.js` 和 `08-account.js` 含 UTF-8 BOM

**证据：**
```
09-vocab.js:  EF BB BF 2F  →  BOM + /
08-account.js: EF BB BF 2F  →  BOM + /
```

`09-vocab.js` 头部注释还有 GBK 乱码（仅注释，不影响语法），但 BOM 本身在 Capacitor Android WebView 中存在风险：

- 部分 Android WebView 版本对 `<script src>` 加载的 JS 文件遇到 BOM 时，会将文件标记为 UTF-16 并解码失败
- 若 `09-vocab.js` 加载失败，以下函数全部 undefined：
  - `normalizeWordEntry` → `12-challenges.js:writeChallengeResultToProgress` 调用时 **ReferenceError**
  - `handleVocabFileImport` / `handleVocabTextImport` → 设置面板点击崩溃
  - `buildSessionWordsSummary` → 游戏结束弹窗崩溃
  - `recordWordResult` → 挑战结果记录失败

**复现路径：**
1. 打开 APK → 登录 → 进入游戏
2. 触发学习挑战（收集单词后随机弹出）
3. 完成挑战 → `writeChallengeResultToProgress` 调用 `normalizeWordEntry` → **崩溃**

**证据：** commit 8952e75 中 `Game.html` 被加了 BOM 头，所有中文文本变成 GBK 乱码。

**已修复：** commit 194a901 从 v1.19.7 恢复干净版本并补入新增结构。

### 问题 3（P1）：`Game.html` 缺少 `btn-export-save` 和 `learning-stats-panel`

**证据：**
```js
// 08-account.js
const btnExportSave = document.getElementById("btn-export-save");      // → null
const learningPanelEl = document.getElementById("learning-stats-panel"); // → null
```

`Game.html` 中只有 `btn-import-save`（已补入），缺少：
- `#btn-export-save` — 存档导出按钮（用户无法导出存档码）
- `#learning-stats-panel` — 学习统计面板（档案页统计区域空白）

代码有 null 检查，不会崩溃，但功能完全失效。

---

### 问题 4（P1）：`normalizeWordEntry` 跨文件依赖

**证据：**
```js
// 12-challenges.js:writeChallengeResultToProgress
const entry = normalizeWordEntry(pr.unique[en]);  // 依赖 09-vocab.js
```

`normalizeWordEntry` 定义在 `09-vocab.js`，被 `12-challenges.js` 直接调用（无 typeof 检查）。若 `09-vocab.js` 因 BOM 加载失败，此处直接抛出 `ReferenceError`，导致挑战完成流程崩溃，`setInputLocked(false)` 无法执行，**玩家输入永久锁死**。

---

### 问题 5（P2）：挑战答错触发无限重试 + 输入锁

**证据：**
```js
// 12-challenges.js:completeLearningChallenge (答错分支)
setTimeout(() => {
    hideLearningChallenge();
    if (typeof setInputLocked === "function") setInputLocked(false);
    startLearningChallenge(retryWord, null, savedOrigin);  // 再次锁定输入
}, 2500);
```

答错流程：
1. `setInputLocked(true)` → 显示纠错 2.5s
2. `setInputLocked(false)` → 立即 `startLearningChallenge` → `setInputLocked(true)`
3. 无限循环，直到答对

这是设计行为，但若步骤 2 因 `normalizeWordEntry` 崩溃而中断，输入将永久锁死。

## 三、修复清单

### Fix 1（P0）：移除 `09-vocab.js` 和 `08-account.js` 的 BOM

```bash
# 用 sed 去除 BOM（EF BB BF）
sed -i 's/^\xef\xbb\xbf//' apk/src/modules/09-vocab.js
sed -i 's/^\xef\xbb\xbf//' apk/src/modules/08-account.js
# 同时修复 09-vocab.js 头部乱码注释（可选，不影响运行）
```

### Fix 2（P1）：在 `Game.html` 补入缺失元素

**`#btn-export-save`** — 加入存档进度弹窗：
```html
<!-- save-progress-modal 的 profile-actions 中 -->
<button class="game-btn game-btn-small" id="btn-export-save">📤 导出存档码</button>
```

**`#learning-stats-panel`** — 加入档案弹窗：
```html
<!-- profile-modal 的 profile-stats 后 -->
<div id="learning-stats-panel" class="profile-stats"></div>
```

### Fix 3（P1）：为 `normalizeWordEntry` 调用加防御

```js
// 12-challenges.js:writeChallengeResultToProgress
const entry = typeof normalizeWordEntry === "function"
    ? normalizeWordEntry(pr.unique[en])
    : { seen: 1, correct: 0, wrong: 0, lastSeen: Date.now(), quality: "new" };
```

## 四、验证方法

1. 打开 `Game.html` → F12 Console → 确认无红色错误
2. 登录 → 收集单词 → 触发挑战 → 答对/答错 → 确认输入不锁死
3. 打开设置 → 点击"导入 .txt/.csv" → 确认无 ReferenceError
4. 打开档案 → 确认学习统计面板有内容
5. 打开存档弹窗 → 确认导出/导入按钮均可点击
6. 死亡 → 确认复盘弹窗正常显示并可关闭
