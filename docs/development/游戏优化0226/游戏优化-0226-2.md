# 游戏优化-0226-2（流程分析 + 执行约束）

## 0. 端到端流程分析（先执行）

本次优化采用严格的交付闭环，避免在 `main` 直接开发导致回滚困难：

1. 清理主仓库工作区（`stash` 或提交）
2. 创建隔离 `worktree` + 功能分支 `feat/optimization-0226`
3. 仅在 `apk/` 目录执行 7 个任务（TDD、逐任务提交）
4. 推送功能分支到远端
5. 在功能分支手动触发/验证 APK CI
6. CI 通过后合入 `main`
7. 推送 `main` 触发正式 Android CI 和 Pages 更新

## 1. 目标

- 提升学习闭环体验（存档迁移、复盘、弱项反馈）
- 提升移动端可玩性（触控区域、输入锁）
- 提升内容扩展能力（词库导入）
- 完成 PWA 离线基础能力

## 2. 执行约束

- 单一开发目录：`apk/`
- 每个任务最小提交，提交信息可追溯
- 每个任务需要可验证产出（测试或构建命令）
- 任何阶段失败，先修复再推进下一阶段

## 3. 任务清单（对应执行计划）

- 任务1：存档导出/导入码
- 任务2：关卡结束复盘
- 任务3：移动端触控优化
- 任务4：词库导入
- 任务5：学习数据可视化 + 弱词清单
- 任务6：成就系统深化
- 任务7：PWA 离线支持
