# 需求分析 - 2026-03-08

## 需求概述

本次需求包含4个主要任务，涉及词库系统、BOSS视觉效果、村庄UI优化和龙蛋获取机制。

## 任务拆分

### 任务1：词库切换与单词记忆曲线优化

**需求描述：**
- 英语学习模式和汉语学习模式都可以切换词库
- 单词随机需要真正有效
- 单词不要一直重复，按规律出现
- 符合记忆曲线设计

**当前问题：**
- manifest.js 已修复，包含所有词库包
- 但单词选择算法可能存在重复问题
- 缺少记忆曲线机制

**技术方案：**
1. 实现间隔重复算法（Spaced Repetition）
2. 记录每个单词的学习状态（新词、复习、熟练）
3. 根据记忆曲线调整单词出现频率
4. 避免短时间内重复相同单词

**涉及文件：**
- `src/modules/09-vocab.js` - 词库系统
- `src/storage.js` - 存储单词学习记录

---

### 任务2：烈焰人BOSS援军视觉优化

**需求描述：**
- 烈焰人召唤的援军目前是几个方块
- 需要设计更美观的形象

**当前问题：**
- 援军视觉效果简陋

**技术方案：**
1. 设计烈焰人援军的视觉形象（小型烈焰人或火焰元素）
2. 添加召唤动画效果
3. 优化援军的渲染

**涉及文件：**
- `src/modules/15-entities-boss.js` - BOSS系统
- `src/modules/14-renderer-entities.js` - 实体渲染

---

### 任务3：村庄建筑UI优化

**需求描述：**
- 床屋的床没看到了，要加上
- 床屋、单词屋、商人屋的图层和文字很乱
- 下面是角色，角色头上一段距离是操作说明
- 商人屋对话框卖材料太挤，增加高度
- 单词屋测验时，单词/汉字出现就要显示读音，不是点击后才出现

**当前问题：**
- 村庄建筑内部UI布局混乱
- 床屋缺少床的视觉元素
- 商人屋UI拥挤
- 单词屋读音显示时机不对

**技术方案：**
1. 床屋：添加床的sprite渲染
2. 统一村庄建筑UI布局：
   - 底部：角色sprite
   - 中部：操作说明文字（角色上方留出间距）
   - 顶部：标题/提示
3. 商人屋：增加对话框高度，优化卖材料列表布局
4. 单词屋：单词/汉字显示时立即播放读音

**涉及文件：**
- `src/modules/18-village.js` - 村庄系统
- `src/modules/14-renderer-main.js` - 主渲染器

---

### 任务4：龙蛋获取机制

**需求描述：**
- 确认宝箱能否开出龙蛋
- 或者在商人屋增加购买龙蛋的功能

**当前问题：**
- 龙蛋获取途径不明确

**技术方案：**
1. 检查当前宝箱掉落表是否包含龙蛋
2. 如果没有，添加龙蛋到高级宝箱掉落表
3. 在商人屋添加龙蛋商品（高价格）

**涉及文件：**
- `config/game.json` - 游戏配置（宝箱掉落表）
- `src/modules/18-village.js` - 村庄系统（商人屋）

---

## 优先级排序

3. **P2 - 任务4**：龙蛋获取（游戏进度相关）
4. **P3 - 任务2**：烈焰人援军视觉（美化，不影响功能）
