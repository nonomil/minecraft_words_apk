# UI一致性分析与优化 - 需求文档

**日期：** 2026-03-04
**状态：** 进行中

## 研究卡片

**研究问题：** 分析游戏UI的一致性问题（弹窗、字体、颜色、大小），输出优化建议

**背景约束：**
- 技术栈：Pure HTML5/CSS3/JavaScript (ES6), Canvas API
- 当前问题：弹窗、字体、颜色、大小不统一，看起来像demo而非成熟产品
- 分析工具：Playwright自动化测试

**期望输出：**
- [x] 使用Playwright自动化分析整个游戏流程
- [ ] 详细的UI元素分析报告（弹窗、按钮、字体、颜色）
- [ ] UI优化建议（设计规范、实施方案）
- [ ] 保存到 docs\plan\UI分析与优化\

**分析范围：**
1. 主菜单界面
2. 设置界面
3. 词汇包选择界面
4. 游戏内UI（HUD、暂停菜单）
5. 游戏结束界面
6. 所有弹窗和对话框

## 分析维度

### 1. 字体一致性
- 字体族（font-family）
- 字体大小（font-size）
- 字体粗细（font-weight）
- 行高（line-height）

### 2. 颜色一致性
- 主色调
- 辅助色
- 文字颜色
- 背景色
- 边框色

### 3. 尺寸一致性
- 按钮大小
- 弹窗大小
- 间距（padding/margin）
- 圆角（border-radius）

### 4. 交互一致性
- 按钮hover效果
- 点击反馈
- 动画效果
- 过渡效果

## 执行计划

1. **Phase 1**：编写Playwright脚本，自动化遍历游戏流程
2. **Phase 2**：提取所有UI元素的样式信息
3. **Phase 3**：分析样式差异，生成对比报告
4. **Phase 4**：输出UI设计规范和优化建议
