# 词汇系统优化 - 分阶段实施计划

> 生成时间：2026-03-05
> 基于：技术设计方案 v2.0

## 执行摘要

本计划将词汇系统优化分为 **4 个阶段**，每个阶段可独立交付，逐步解决问题。

**总体时间线**：8-10 周
**核心原则**：
- 每个阶段独立可测试、可交付
- 优先解决高风险问题
- 保持向后兼容
- 持续集成和测试

---

## 阶段 1：紧急修复（1-2 周）

### 目标
解决 P0 级问题，确保系统稳定运行，为后续迁移做准备。

### 任务清单

#### Task 1.1：LocalStorage 容量优化（3 天）
**优先级**：P0
**负责模块**：`src/storage.js`

**子任务**：
- [ ] 1.1.1 实现数据压缩（使用 LZString）
  - 压缩账户数据
  - 压缩进度数据
  - 预期减少 50-70% 存储空间

- [ ] 1.1.2 清理冗余数据
  - 删除 `wordStats`（已弃用）
  - 合并 `learnedWords` 和 `packProgress.unique`
  - 清理过期备份（保留最近 3 个）

- [ ] 1.1.3 添加容量监控
  - 实时监控 LocalStorage 使用量
  - 达到 80% 时警告用户
  - 提供清理建议

**验收标准**：
- ✅ 单账户数据 < 100KB（当前 ~200KB）
- ✅ 支持至少 10 个账户不溢出
- ✅ 有容量警告机制

**风险**：
- 压缩/解压可能影响性能 → 缓解：仅在保存/加载时压缩

## 阶段 2：核心优化（2-3 周）

- [ ] 2.1.1 创建 IndexedDB Schema

  - 实现版本管理和升级逻辑

- [ ] 2.1.2 实现 CRUD 操作

  - 实现批量操作
  - 实现事务管理

- [ ] 2.1.3 实现缓存层

  - 5 分钟 TTL
  - 缓存失效策略

**验收标准**：
- ✅ IndexedDB 初始化成功

- ✅ 缓存命中率 > 70%

**风险**：

## 质量保证

### 测试策略

**单元测试**：
- 覆盖率目标：> 80%
- 关键模块：100% 覆盖
- 工具：Jest

**集成测试**：
- 端到端流程测试
- 数据迁移测试
- 导入/导出测试
- 工具：Playwright

**性能测试**：
- 加载性能：首屏 < 2s
- 运行性能：操作响应 < 100ms
- 存储性能：批量操作 < 5s
- 工具：Lighthouse + 自定义脚本

**兼容性测试**：
- 浏览器：Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 移动端：iOS Safari 14+, Chrome Android 90+
- 工具：BrowserStack

### 代码审查

**审查流程**：
1. 开发完成后自我审查
2. 提交 Pull Request
3. 同行审查（至少 1 人）
4. 通过后合并

**审查重点**：
- 代码质量和可读性
- 性能影响
- 安全性（XSS、注入等）
- 测试覆盖率

### 持续集成

**CI 流程**：
1. 代码提交触发 CI
2. 运行 Linter（ESLint）
3. 运行单元测试
4. 运行集成测试
5. 生成测试报告
6. 通过后允许合并
