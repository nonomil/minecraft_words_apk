# 第一阶段奖励闭环验收清单

> 日期：2026-03-06

> 配套文档：`docs/plan/游戏优化设计/2026-03-06-第一阶段奖励闭环实施计划.md`
> 当前状态：待执行

---

## 1. 本清单目的

本清单用于在第一阶段奖励闭环开发完成后，按统一标准进行验收，避免出现“功能看起来有了，但实际体验不完整”的情况。

重点验证以下四类问题：

1. 学习行为是否真的能触发即时奖励。
2. 奖励是否真的影响到玩家能感知的游戏结果。
3. 小学低年级模式下是否明显降低了挫败感。
4. 新逻辑是否破坏了现有主流程。

---

## 2. 验收范围

本轮验收只覆盖第一阶段定义的内容：

- 奖励配置接入
- 挑战答对后的即时奖励
- 宝箱与学习表现绑定
- 学习连胜与短时增益
- 小学低年级减压调优
- UI 即时反馈与回归兼容性

本轮不作为硬验收项的内容：

- 幼儿园模式重设计
- 初中深度模式
- 新群系/新敌人大扩展
- 大规模经济系统重构

---

## 3. 验收前准备

执行验收前，应完成以下准备：

- [ ] 已同步最新代码与配置文件
- [ ] 奖励闭环相关改动已合并到可运行分支
- [ ] 游戏可以正常启动
- [ ] 已知与本次任务无关的旧问题已记录
- [ ] 验收人员知道本轮目标用户为“小学低年级”

建议准备两种验证方式：

- 自动化：运行最相关的 e2e / smoke 用例
- 手工：进入游戏进行完整体验回归

---

## 4. 功能验收项

### 4.1 奖励配置与全局状态

**目标：** 配置加载稳定，奖励逻辑有统一入口。

- [ ] `config/reward-loops.json` 能成功加载
- [ ] 配置加载失败时有默认兜底，不导致启动崩溃
- [ ] 全局状态结构可用，不出现 `undefined` / `null` 链式错误
- [ ] 未触发奖励逻辑时，游戏仍保持原流程

**通过标准：**

- 游戏启动正常
- 控制台无明显配置读取异常
- 不开启奖励时主循环无回归问题

### 4.2 挑战答对后的即时奖励

**目标：** 玩家答对后，能立刻感知到奖励。

- [ ] 答对题目后，立刻出现奖励反馈文案
- [ ] 奖励不是只有分数变化，还能体现为护盾/掉落/短时增益等效果
- [ ] 奖励提示不会挡住核心操作区
- [ ] 连续答题时奖励状态能持续刷新
- [ ] 答错时不会错误触发正向奖励

**通过标准：**

- 玩家能明确知道“答对了，所以我得到了什么”

### 4.3 宝箱与学习表现绑定

**目标：** 玩家能理解“学得好，箱子更好”。

- [ ] 最近学习表现能影响后续宝箱/掉落结果
- [ ] 学习表现加成不会永久叠满不消失
- [ ] 宝箱升级或掉落加成有 UI 提示说明来源
- [ ] 未触发学习加成时，普通宝箱仍正常工作
- [ ] 高品质奖励仍保留随机性，没有完全失衡

**通过标准：**

- 玩家能感知到奖励提升，同时经济平衡未明显被破坏

### 4.4 学习连胜与短时增益

**目标：** 连续答对能形成持续动力，但不会造成脏状态。

- [ ] 连续答对达到阈值后，正确触发对应奖励
- [ ] 奖励到期后能自动清理
- [ ] 连胜中断后按预期降级，不出现负值或异常重置
- [ ] HUD 能显示当前连胜与奖励状态
- [ ] 不与已有战斗 buff 产生明显冲突

**通过标准：**

- 连胜奖励逻辑稳定，体验上“有激励但不复杂”

### 4.5 小学低年级减压调优

**目标：** 前期压力下降，但挑战感还在。

- [ ] 低年级默认配置下，前期答题时间更宽松
- [ ] 低年级默认配置下，答错惩罚更轻
- [ ] 前 5~10 分钟更容易获得首次正向奖励
- [ ] UI 鼓励文案更易懂、更正向
- [ ] 整体体验不是“没有挑战”，而是“更愿意继续尝试”

**通过标准：**

- 小学低年级用户在前期更容易进入状态，不因一两次失败直接退出

---

## 5. 回归检查项

以下内容必须作为回归项检查，防止新奖励逻辑破坏旧系统：

- [ ] 游戏主循环正常：移动、跳跃、攻击、受伤、死亡都不异常
- [ ] 原有学习挑战仍能正常弹出、作答、结算
- [ ] Word Gate 不因奖励状态接入而失效
- [ ] Word Match Revival 不因新奖励状态出现卡死或重复触发
- [ ] 宝箱本身仍可正常生成、开启、掉落
- [ ] 单词收集、发音播放、词卡展示正常
- [ ] 存档/本地设置不因新增状态而损坏

---

## 6. 推荐验收流程

建议按以下顺序执行：

1. 先做启动与基础稳定性检查
2. 再做挑战奖励链路检查
3. 再做宝箱加成与连胜检查
4. 再做小学低年级前期体验检查
5. 最后做整局回归体验

推荐至少覆盖以下场景：

- 场景 A：正常答对 1 次，检查即时奖励
- 场景 B：连续答对 3~5 次，检查连胜奖励
- 场景 C：在学习加成生效期间开启宝箱，检查品质提升
- 场景 D：故意答错，检查是否不会误触发奖励
- 场景 E：低年级默认配置下体验前 10 分钟，检查挫败感是否下降

## 8. 通过门槛

建议把以下条件作为“可进入下一阶段”的最低门槛：

- [ ] 即时奖励链路可稳定复现
- [ ] 宝箱加成链路可稳定复现
- [ ] 连胜状态不会产生脏数据
- [ ] 小学低年级前期体验明显优于旧版本
- [ ] 不存在阻断主流程的严重 bug

如果以上任一项不满足，则本轮不应宣告完成。
