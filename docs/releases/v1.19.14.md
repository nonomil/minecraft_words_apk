# Release Notes - v1.19.14

**Release Date**: 2026-03-06

## 🐛 Bug Fixes

This release fixes critical issues with Chinese mode display and improves the language selection user experience.

### ✨ Fixed Issues

#### Chinese Mode Display Fix
- **Problem**: In Chinese mode, word items still displayed English words instead of Chinese characters
- **Root Cause**: `updateWordUI()` function was hardcoded to always display `wordObj.en` (English)
- **Solution**: Updated to use `BilingualVocab.getDisplayContent()` which returns correct content based on language mode
  - Chinese mode: primaryText = Chinese characters, secondaryText = English
  - English mode: primaryText = English, secondaryText = Chinese
- **Impact**: Chinese learners now see Chinese characters when collecting word items

#### Language Selection UX Improvement
- **Problem**: Separate language selection modal popup felt disconnected from main flow
- **Solution**: Integrated language selection directly into login screen
  - Removed `language-mode-onboarding-modal` popup
  - Added language selection buttons (🇬🇧 英语学习 / 🇨🇳 汉字学习) to login screen
  - Automatically hides after user makes selection
- **Impact**: Smoother onboarding experience, fewer modal popups

#### Test Suite Fixes
- **Problem**: E2E tests were accessing `window.settings` directly, which returned `undefined`
- **Root Cause**: `settings` is a local variable in module scope, not exposed to window
- **Solution**: Updated all tests to use `window.MMWG_TEST_API.getState().settings`
- **Impact**: All 6 Chinese mode tests now pass (previously 4 failed)

### 🔧 Technical Changes

**Modified Files**:
