# 项目初始化

分析项目代码结构，生成 `claude-template.md` 文档，便于快速理解项目架构。

## 执行步骤

### 1. 检查现有文档
检查项目根目录是否已有 `claude-template.md`：
- 已有 → 询问用户是更新还是重新生成
- 没有 → 继续分析

**注意**：不会修改 `claude.md`（用户全局指令），只生成 `claude-template.md`（项目结构参考）

### 2. 分析项目结构
```bash
# 查看目录结构
ls -la
# 或 Windows
dir
```

识别：
- 项目类型（前端/后端/全栈/库等）
- 技术栈（语言、框架）
- 主要目录结构
- 入口文件

### 3. 识别模块
根据目录结构识别独立模块，判断标准：
- 独立的功能职责
- 相对独立的代码目录
- 有明确的边界

### 4. 分析关键文件
读取关键文件了解项目：
- package.json / requirements.txt / go.mod 等依赖文件
- 配置文件
- 入口文件
- 各模块的主要文件

### 5. 生成项目 claude-template.md
在项目根目录创建 `claude-template.md`：

```markdown
## 项目概览
[一句话描述项目用途]

## 技术栈
- 语言：
- 框架：
- 主要依赖：

## 目录结构
```
[主要目录树]
```

## 模块说明
| 模块 | 路径 | 职责 |
|------|------|------|
| xxx | src/xxx | xxx |

## 开发注意事项
[重要的注意事项]

## 当前任务
[空，后续填写]

## 最近修改
[空，后续填写]
```

### 6. 生成模块 claude-template.md（可选）
对于复杂模块，在模块目录下创建 `claude-template.md`：

```markdown
## 模块职责
[这个模块做什么]

## 关键文件
| 文件 | 作用 |
|------|------|
| xxx | xxx |

## 依赖关系
- 依赖：[依赖的其他模块]
- 被依赖：[依赖此模块的其他模块]

## 核心逻辑
[关键流程说明]

## 注意事项
[开发时需要注意的点]

## 最近修改
[空，后续填写]
```

### 7. 确认结果
告知用户：
- 已生成的文档列表
- 建议检查并补充的内容

## 判断是否需要模块 claude-template.md

| 条件 | 是否生成 |
|------|---------|
| 模块代码量大（>10个文件） | 是 |
| 逻辑复杂 | 是 |
| 模块代码量小且简单 | 否，在项目 claude-template.md 中说明即可 |

## 注意事项
- 文档要简洁，避免冗余
- 只记录对理解项目有帮助的信息
- 不要复制代码到文档，只描述职责和逻辑
- 如果项目已有 README.md，可以参考但不要重复
- **重要**：不会修改 `claude.md`，只生成 `claude-template.md` 作为参考
