# 03｜AI 资料调研搜索阶段模板：多源搜索 → 交叉验证 → 整合输出

> 通用模板：适用于技术选型、方案调研、新领域探索、结论验证。
> 工具组合：Claude 网页版联网搜索 + Google 深度研究 + 秘塔/MiroMind + Opus 整合。
> 将 `[RESEARCH_*]` 占位符替换为实际信息后使用。

---

## 前置信息（填写后开始）

| 变量 | 说明 | 示例 |
|------|------|------|
| `[RESEARCH_TOPIC]` | 调研主题 | `PWA 离线缓存最佳实践` |
| `[RESEARCH_GOAL]` | 调研目标 | `确定 Service Worker 缓存策略` |
| `[OUTPUT_DIR]` | 输出目录 | `docs/research/` |
| `[DEADLINE]` | 需要结论的时间 | `2026-02-26` |
| `[DEPTH]` | 调研深度 | `快速（1-2小时）/ 标准（半天）/ 深度（1天）` |

---

## 完整流程概览

```
Phase 0   明确调研问题（避免调研跑偏）
Phase 1   Claude 网页版深度联网搜索（10轮迭代）
Phase 2   Google 深度研究模式（生成搜索方向）
Phase 3   多工具并行搜索（秘塔 / MiroMind / 其他）
Phase 4   Opus 整合多源文档 → 生成统一调研报告
Phase 5   结论验证与落地性评估
Phase 6   输出最终调研文档 → 移交计划阶段
```

## Phase 1：Claude 网页版深度联网搜索（10轮迭代）

**工具：** Claude 网页版（claude.ai），开启联网搜索功能

**初始 Prompt（粘贴到 Claude 网页版）：**

```
## 深度调研任务：[RESEARCH_TOPIC]

**调研目标：** [RESEARCH_GOAL]

**核心问题：**
1. [子问题1]
2. [子问题2]
3. [子问题3]

---

请按以下方式进行深度探索：

**搜索策略：**
深度探索本问题，对每个结论进行联网搜索，确认正确性。
每次搜索获得信息后，都去审视当前所获取信息的不足之处，当前结论是否正确，并发起新一轮搜索。
整个过程至少重复 10 轮。

**分析要求：**
- 辩证分析，确认正确性，确认有没有问题
- 区分：官方文档结论 / 社区经验 / 个人观点 / 过时信息
- 标记每个结论的可信度：✅ 有官方来源 / ⚠️ 社区经验 / ❌ 待确认
- 识别业内通用方法和成熟方案
- 对比不同方案的优缺点

**输出要求：**
我需要一个最终可落地、工程上可行的方案。
请在每轮搜索后更新结论，最终输出：
1. 核心结论（有来源标注）
2. 推荐方案（含理由）
3. 备选方案（含权衡）
4. 仍不确定的问题（需进一步验证）
5. 参考资料列表（含 URL）
```

| 10 | | | | |

**保存 Claude 输出到：** `[OUTPUT_DIR]YYYY-MM-DD-[RESEARCH_TOPIC]-claude.md`

## Phase 4：Opus 整合多源文档

**工具：** Antigravity / Kiro，**Plan Mode + Opus 模型**

**Prompt：**

```
我有来自多个 AI 工具的调研文档，请帮我整合：

## 文档列表
1. Claude 联网搜索：[OUTPUT_DIR]xxx-claude.md
2. 秘塔搜索：[OUTPUT_DIR]xxx-metaso.md
3. MiroMind：[OUTPUT_DIR]xxx-miromind.md
[其他文档...]

## 整合要求

**核心问题：** [RESEARCH_GOAL]

请按以下方式整合：

1. **结论一致性分析**
   - 哪些结论在多个来源中一致？（高可信度）
   - 哪些结论只有单一来源？（需标注）
   - 哪些结论在不同来源中矛盾？（需辩证分析）

2. **辩证分析**
   对每个矛盾点：
   - 分析各方观点的依据
   - 判断哪个更可信（基于来源权威性、时效性、逻辑性）
   - 给出明确结论，不要模糊两可

3. **落地性评估**
   对每个推荐方案：
   - 工程可行性（1-5分）
   - 实施复杂度（低/中/高）
   - 维护成本（低/中/高）
   - 适用场景

4. **输出格式**
   生成整合报告，保存到：
   [OUTPUT_DIR]YYYY-MM-DD-[RESEARCH_TOPIC]-final.md

   报告结构：
   - 执行摘要（3-5句话，核心结论）
   - 推荐方案（含理由和落地步骤）
   - 备选方案（含权衡分析）
   - 仍不确定的问题（需实验验证）
   - 参考资料（按可信度排序）
```

**整合报告标准结构：**

```markdown
# [RESEARCH_TOPIC] 调研报告

> 生成日期：YYYY-MM-DD
> 来源：Claude 联网 + 秘塔 + MiroMind + [其他]
> 整合：Claude Opus

## 执行摘要
[3-5句话，核心结论，可直接用于决策]

## 推荐方案

**核心优势：** [列表]
**主要限制：** [列表]
**落地步骤：**

[同上结构]

|--------|-----------|-----------|----------|------|
```

## 注意事项

1. **信息时效性**：AI 搜索结果可能包含过时信息，优先看官方文档日期
2. **来源标注**：每个结论必须标注来源，"AI 说的"不是来源
3. **矛盾是正常的**：不同工具给出矛盾结论时，不要取平均，要辩证分析
4. **实验验证优先**：调研结论再好，关键决策前必须小规模实验验证
5. **调研不是目的**：调研是为了做决策，超过 1 天没有结论就停止，用现有信息决策
6. **避免分析瘫痪**：信息足够做决策时就停止调研，完美是好的敌人
