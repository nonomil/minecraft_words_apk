# 项目迁移指南 — Claude Code + Codex 协作配置

> 本文档说明如何将当前项目的 Claude Code + Codex 协作配置迁移到其他项目或其他电脑。

---

## 0. 先安装（Codex MCP + Cartographer）

在开始迁移前，先确保两项基础能力已就绪：
- Codex MCP（用于代码执行与改造）
- Cartographer 插件（用于生成 `docs/CODEBASE_MAP.md`）

### 0.1 安装 Codex MCP

```bash
claude mcp add codex -s user -- codex mcp-server
claude mcp list
```

预期：`claude mcp list` 中出现 `codex` 且状态为已连接。

### 0.2 安装 Cartographer 插件

如果在 Claude Code 交互会话内执行：

```text
/plugin marketplace add kingbootoshi/cartographer
/plugin install cartographer
```

如果在系统命令行（PowerShell/CMD）执行：

```bash
claude plugin marketplace add kingbootoshi/cartographer
claude plugin install cartographer@cartographer-marketplace
claude plugin list
```

预期：`claude plugin list` 中存在 `cartographer@cartographer-marketplace`，状态为 `enabled`。

### 0.3 Windows 文件锁豁免说明（安装失败时）

若安装插件时报 `EBUSY: resource busy or locked`（常见于 Windows），可按以下方式处理：
- 先重启终端或重启系统后重试安装命令
- 不阻塞当前迁移：可先完成文档与流程迁移，插件安装后补跑 `/cartographer`

## 场景 1：迁移到新项目（同一电脑）

### Step 1：复制 .claude 目录

```bash
# 从源项目复制
cp -r 源项目/.claude 目标项目/

# Windows PowerShell
Copy-Item -Recurse 源项目\.claude 目标项目\.claude
```

### Step 2：复制根目录配置文件

```bash
# 复制这些文件到目标项目根目录
cp 源项目/claude.md 目标项目/
cp 源项目/AGENTS.md 目标项目/
```

> 注意：工作流文档（claude-workflow-*.md）和全局约束（claude-workflow-constants.md）已包含在 `.claude/` 目录中，Step 1 复制时已一并带过去，无需单独复制。

### Step 3：修改项目特定配置

编辑 `目标项目/.claude/settings.local.json`，修改以下部分：

```json
{
  "permissions": {
    "allow": [
      // 改为目标项目的实际命令
      "Bash(git init)",
      "Bash(git add 目标项目/)",
      "Bash(git commit -m \"feat: ...\")"
    ]
  },
  "hooks": {
    // Hook 脚本路径保持不变（相对路径）
    // 但如果有绝对路径，需要改为相对路径
  }
}
```

### Step 4：验证 Hook 生效

```bash
cd 目标项目
python .claude/scripts/auto_checkpoint_commit.py --dry-run
```

预期输出：
```
[DRY-RUN] 将执行提交: checkpoint: HH:MM
```

## 场景 2：迁移到新电脑

### Step 1：克隆项目

```bash
git clone <repo-url> 项目目录
cd 项目目录
```

### Step 2：验证 .claude 目录完整性

```bash
# 检查关键文件是否存在
ls -la .claude/settings.local.json
ls -la .claude/scripts/
ls -la .claude/templates/
```

如果缺失，从源电脑复制（见场景 1 Step 1）。

### Step 3：检查全局 Hook 配置

编辑 `~/.claude/settings.json`（用户全局配置），确保包含：

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash(rm -rf*)",
        "hooks": [
          {
            "type": "command",
            "command": "python C:/Users/[用户名]/.claude/hooks/block-delete.py"
          }
        ]
      }
    ]
  }
}
```

**Windows 路径注意**：使用绝对路径，格式为 `C:/Users/[用户名]/...`（正斜杠）。

### Step 4：验证 Python 环境

```bash
python --version  # 需要 3.8+
python -c "from pathlib import Path; print('OK')"
```

### Step 5：测试 Hook

```bash
cd 项目目录
python .claude/scripts/auto_checkpoint_commit.py --dry-run
```

## 场景 3：从旧项目升级（无 .claude 目录）

### 前置条件

旧项目结构：
```
项目根目录/
├── workflow-constants.md
├── docs/templates/
│   ├── largebase-scan-pack-template.md
│   ├── parallel-impact-scope-template.md
│   └── ...
└── claude.md
```

### 升级步骤

#### Step 1：创建 .claude 目录结构

```bash
mkdir -p .claude/workflows
mkdir -p .claude/scripts
mkdir -p .claude/templates
mkdir -p .claude/skills
mkdir -p .claude/reference
```

#### Step 2：移动文件

```bash
# 移动常量文件
mv workflow-constants.md .claude/workflows/claude-workflow-constants.md

# 移动模板
mv docs/templates/largebase-scan-pack-template.md .claude/templates/
mv docs/templates/largebase-scan.schema.json .claude/templates/
mv docs/templates/largebase-scan.sample.json .claude/templates/
mv docs/templates/parallel-impact-scope-template.md .claude/templates/
```

#### Step 3：创建 Hook 脚本

从源项目复制以下脚本到 `.claude/scripts/`：
- `auto_checkpoint_commit.py`
- `append_changelog_draft.py`
- `pre_merge_scope_guard.py`

#### Step 4：创建项目级 Hook 配置

创建 `.claude/settings.local.json`（参考本文档下方的"配置文件模板"）。

#### Step 5：更新所有文档引用

使用 Python 脚本批量替换（**禁止用 PowerShell Get-Content/Set-Content**）：

```python
from pathlib import Path
import re

# 替换规则
replacements = {
    r'workflow-constants\.md': '.claude/workflows/claude-workflow-constants.md',
    r'docs/templates/': '.claude/templates/',
}

# 遍历所有 Markdown 和 JSON 文件
for file_path in Path('.').rglob('*'):
    if file_path.suffix not in ['.md', '.json', '.txt']:
        continue
    if '.git' in file_path.parts:
        continue

    try:
        content = file_path.read_text(encoding='utf-8')
        original = content

        for pattern, replacement in replacements.items():
            content = re.sub(pattern, replacement, content)

        if content != original:
            file_path.write_text(content, encoding='utf-8')
            print(f"Updated: {file_path}")
    except Exception as e:
        print(f"Skipped {file_path}: {e}")
```

#### Step 6：验证引用

```bash
# 检查是否还有旧路径
rg "workflow-constants\.md|docs/templates/" --type md --type json
```

预期：无输出。

#### Step 7：提交

```bash
git add .claude/ claude.md AGENTS.md
git commit -m "refactor: migrate to .claude directory structure"
```

## 常见问题

### Q1：Hook 没有生效

**检查清单**：
1. `.claude/settings.local.json` 是否存在且格式正确？
   ```bash
   python -m json.tool .claude/settings.local.json
   ```

2. 脚本文件是否存在且可执行？
   ```bash
   ls -la .claude/scripts/auto_checkpoint_commit.py
   ```

3. Python 是否可用？
   ```bash
   python --version
   ```

4. Claude Code 是否已重新加载项目？
   - 关闭 VSCode，重新打开项目

### Q2：文件编码乱码

**原因**：PowerShell `Get-Content`/`Set-Content` 默认用系统编码（GBK）。

**解决**：
1. 从 git 恢复：
   ```bash
   git show HEAD~1:文件路径 > 文件路径
   ```

2. 使用 Python 读写文本文件：
   ```python
   from pathlib import Path
   content = Path('file.md').read_text(encoding='utf-8')
   Path('file.md').write_text(content, encoding='utf-8')
   ```

### Q3：引用路径错误

**症状**：文档里的链接指向不存在的文件。

**检查**：
```bash
# 查找所有引用
rg "\.claude/templates|\.claude/workflows/claude-workflow-constants" --type md

# 验证文件是否存在
ls -la .claude/templates/
ls -la .claude/workflows/claude-workflow-constants.md
```

### Q4：跨电脑同步

**推荐方案**：
1. 将 `.claude/` 目录提交到 git
2. 在新电脑上 `git clone` 后，自动获得所有配置
3. 仅需修改全局 `~/.claude/settings.json` 中的绝对路径
